        Gate.io Market Scanner — Documentation
        Complete reference for all features, signals, calculations, and risk rules.
        1. Overview 2. Scanner 3. Indicators 4. Signal Generation 5. Grading & Probability 6. Risk Management 7. Trade Journal 8. Capital & Compounding 9. Breakout Scanner 10. Price Monitor 11. Evaluation Metrics 12. Action Plan 13. AI Analysis

        1. Overview

        This app is a real-time cryptocurrency market scanner and trade journal built for Gate.io. It continuously scans hundreds of spot and futures pairs, scores them using technical analysis, auto-logs high-quality setups to a trade journal, monitors open positions, and compounds capital over time.

        How It Works — End to End

        • Scanner fetches OHLCV data for the top-volume USDT pairs on Gate.io every 5 minutes.
        • Each coin is scored using EMA crossover, RSI, ADX, MACD, volume, and trend indicators.
        • Signals with a valid entry/stop/target are auto-logged to the Trade Journal — subject to the daily 1% risk budget.
        • The Price Monitor checks live prices every 60 seconds and auto-closes trades when stop-loss or take-profit is hit.
        • Capital compounds — each closed trade's PnL is added to or subtracted from the running capital balance, which then adjusts position sizes for future trades.
        • The Evaluation tab shows comprehensive performance analytics including Sharpe ratio, Profit Factor, grade breakdown, and an AI-generated analysis.
        All data is stored locally in logs/journal.json and logs/capital.json. No exchange account is required — this is a paper trading tracker.

        2. Market Scanner

        The scanner fetches the top-volume USDT pairs from Gate.io and analyzes each one in parallel using 8 worker threads.

        Configuration

        ParameterDefaultDescription
        Timeframe4hCandle timeframe for all indicator calculations
        Candles fetched300Historical bars per coin (min 220 required for EMA-200)
        Min volume500,000 USDT24h volume filter — low-volume coins are excluded
        Max pairs80 (spot), 50 (futures)Top N pairs by volume to analyze per scan
        Scan interval5 minutesAuto-scan repeats continuously in background
        Worker threads8Parallel analysis threads for speed

        Market Types

        • Spot — Regular spot markets (e.g., BTC/USDT). Always 1× leverage.
        • Futures (PERP) — Perpetual futures (e.g., BTC/USDT:USDT). Supports leverage. Includes funding rate data.

        Multi-Timeframe (MTF) Alignment

        For each coin, a daily (1D) candle set is also fetched and cached for 30 minutes. The daily EMA-200 position determines the macro trend:

        Daily pct = (price − EMA200_daily) / EMA200_daily × 100 UPTREND if pct > 0.5% | DOWNTREND if pct < −0.5% | SIDEWAYS otherwise

        A signal is MTF aligned when the 4h signal direction matches the daily trend (e.g., BULLISH signal + daily UPTREND). Grade A requires MTF alignment.

        3. Technical Indicators

        All indicators are computed from 4h OHLCV data using custom implementations in indicators.py. No external TA library is used.

        IndicatorPeriodRole
        EMA Fast9Short-term momentum — crossover trigger
        EMA Slow21Medium-term trend — crossover trigger
        EMA Trend200Long-term trend filter — price above/below determines bias
        RSI14Momentum oscillator. Overbought ≥65, Oversold ≤35 (tighter than classic 70/30 for crypto)
        MACD12/26/9Trend momentum. Uses histogram (MACD line − Signal line)
        ATR14Average True Range — measures volatility, used for stop placement and target sizing
        ADX14Trend strength. >25 = trending, >40 = strong trend
        Bollinger Bands20 / 2σVolatility envelope. Width compression signals potential breakout
        Volume Ratio20-bar avgCurrent volume ÷ 20-bar SMA. >1.5 = above average, >3.0 = spike
        OBV—On-Balance Volume. Rising OBV with flat price = accumulation (bullish divergence)

        EMA Crossover Detection

        Bullish cross: EMA9[-1] > EMA21[-1] AND EMA9[-2] ≤ EMA21[-2] Bearish cross: EMA9[-1] < EMA21[-1] AND EMA9[-2] ≥ EMA21[-2]

        The crossover checks the previous bar ([-2]) to confirm the cross is fresh, not already established.

        4. Signal Generation & Scoring

        Each coin is scored using a weighted multi-component system. The final score (0–1) determines signal direction and trade quality.

        Component Weights

        ComponentWeightMax Score Condition
        EMA Crossover30%Fresh cross this bar = 1.0; fast > slow (no cross) = 0.6
        Trend Filter (EMA-200)20%Price above/below EMA-200, scaled by distance (0.5–1.0)
        ADX Strength15%ADX ≥40 = 1.0; ADX ≥25 = 0.5–0.83; ADX <25 = 0
        RSI Condition15%RSI 40–60 (ideal zone) = 1.0; oversold <35 = 0.9; overbought >65 = 0
        Volume Confirmation12%Ratio ≥2.0 = 1.0; ratio ≥1.5 = 0.5–1.0; ratio <1.5 = 0–0.33
        MACD Momentum8%Positive + rising = 1.0; positive only = 0.7; turning up = 0.4

        Signal Direction

        BULLISH if bull_score ≥ bear_score AND bull_score > 0.20 BEARISH if bear_score > bull_score AND bear_score > 0.20 NEUTRAL otherwise

        Entry, Stop & Target

        Entry/stop/target levels use ATR-based placement, scaled by grade:

        Stop = entry − ATR × 1.5 (BULLISH) | entry + ATR × 1.5 (BEARISH) Target = entry + ATR × multiplier (Grade A: 4.0×, B: 3.0×, C: 2.5×)

        This gives approximate R:R ratios of 2.7:1 (Grade C), 2:1 (Grade B), and 2.7:1 (Grade A).

        5. Grading & Probability

        Each signal receives a grade (A/B/C) and estimated win probability based on score, MTF alignment, ADX, and sentiment.

        Win Probability Calculation

        Base probability = linear interpolation: score 0.65 → 50%, score 1.0 → 68% + MTF aligned (daily trend matches 4h signal): +5% + Strong sentiment confluence: +3% + ADX > 30: +3% Maximum cap: 72%

        Grade Thresholds

        GradeMin ProbabilityMin ScoreMTF RequiredBase RiskLeverage
        A≥ 60%≥ 0.75Yes (strict)1.5%3× (futures)
        B≥ 54%≥ 0.65No1.0%2× (futures)
        C< 54%< 0.65No0.5%1× (spot only)
        Grade A requires ALL three gates: probability ≥60%, score ≥0.75, AND MTF aligned. A high-scoring counter-trend trade maxes out at Grade B.

        Expected Value (EV)

        EV = P(win) × (R:R × risk%) − P(loss) × risk% Example: P=62%, R:R=3.0, risk=1.5% EV = 0.62 × 4.5% − 0.38 × 1.5% = +2.22% per trade

        EV represents the expected return per trade as a percentage of capital. Positive EV = mathematically profitable long-term. Negative EV = avoid the setup.

        6. Risk Management

        Daily 1% Loss Cap

        The most important protection rule: total realized losses cannot exceed 1% of current capital per UTC day.

        Daily cap (USDT) = capital × 1% Per-trade budget = Daily cap ÷ MAX_CONCURRENT_TRADES (default 2) = 0.5% per trade maximum
        • If today's losses reach the 1% cap, no new trades are logged until the next UTC day.
        • The daily budget is split across a maximum of 2 simultaneous trades — each can lose at most 0.5%.
        • The progress bar in the Journal stats row shows how much of today's budget has been consumed.
        • The budget resets automatically at UTC midnight.
        Example: Capital = $10,000 → Daily cap = $100 → Per-trade max loss = $50 (0.5%).

        Position Sizing

        risk_amount = capital × risk_pct / 100 qty = (risk_amount / |entry − stop|) × leverage position_value = qty × entry margin_required = position_value / leverage MAX margin = capital × 20% (hard cap to prevent oversized positions)

        Dynamic Risk Adjustment

        The base risk percentage is further adjusted by recent performance:

        ConditionMultiplier
        Win streak ≥ 3× 1.20 (scale up)
        Loss streak ≥ 2× 0.75 (scale down)
        Drawdown > 10%× 0.50 (emergency mode)

        These multipliers are applied to the grade base risk, then capped at the per-trade budget (0.5%).

        2R Loss Cap Per Trade

        Individual trade losses are capped at 2× the risk amount (2R) regardless of how far price moves against the position. This prevents catastrophic losses from extreme price gaps.

        Max loss per trade = risk_amount × 2 If raw_pnl < −2R → pnl capped at −2R, risk_exceeded flag = true

        7. Trade Journal

        Every signal that has a valid entry, stop-loss, and take-profit level is automatically logged to the journal — subject to the daily risk budget and deduplication rules.

        Auto-Logging Rules

        • Signal must have entry, stop, AND target prices.
        • No existing OPEN trade for the same symbol (prevents duplicate positions).
        • Daily loss cap must not be reached.
        • Max 2 concurrent OPEN trades allowed at once.
        • The dynamic_risk_pct is overridden with the daily budget per-trade allocation (0.5%) at log time.

        Trade Status Lifecycle

        OPEN → WIN (price hit take-profit) OPEN → LOSS (price hit stop-loss) OPEN → CANCELLED (manually cancelled)

        Fields Logged Per Trade

        FieldDescription
        symbol, signal, timeframeCoin, direction (BULLISH/BEARISH), 4h
        grade, score, probability, expected_valueSignal quality metrics at entry time
        entry, stop, targetPrice levels for the trade
        risk_reward(target − entry) / (entry − stop)
        dynamic_risk_pct, risk_amountRisk % and USDT amount at risk
        position_value, margin_requiredFull notional exposure and locked margin (capped at 20% capital)
        qty, leveragePosition quantity and leverage used
        capital_at_logCapital snapshot when trade was entered
        rsi, adx, mtf_aligned, daily_trendMarket condition indicators at entry
        exit_price, exit_time, pnlFilled when trade closes
        risk_exceededTrue if loss exceeded 2R

        Manual Log

        Use the "Log Trade" button on any scanner result to manually add a trade to the journal. The same auto-logging rules apply.

        8. Capital & Compounding

        The bot tracks a compounding capital balance. Each closed trade's PnL adjusts the capital, which then changes position sizes for all future trades.

        How Capital Updates

        pnl_usdt = position_value × pnl_pct / 100 new_capital = capital + pnl_usdt peak = max(peak, new_capital)

        Drawdown

        drawdown_pct = (capital / peak − 1) × 100 [negative value means drawdown] growth_pct = (capital / initial − 1) × 100

        Capital Reconciliation

        Capital can drift from the journal's true sum due to edge cases (zero position size, interrupted updates). The Reconcile Capital button rebuilds capital from scratch using journal entries:

        running_capital = initial for each closed trade (sorted by exit_time): running_capital += trade.pnl capital.json is updated with the correct value

        This also runs automatically on every server startup to ensure capital is always in sync.

        Set Capital

        Click the ✏️ icon next to "Capital" in the Journal stats to manually set the capital amount (e.g., after a deposit). This resets both capital and initial to the new value and resets the peak.

        9. Breakout Scanner

        The Breakout filter identifies coins in a pre-breakout compression state — before the EMA crossover happens. These are setups where a move is building up pressure and may accelerate soon.

        Breakout Score Components

        SignalWeightCondition
        BB Squeeze+0.25Bollinger Band width < 80% of its own 20-bar average (volatility compression)
        Strong Squeeze+0.10Squeeze intensity > 50% (severely compressed)
        ATR Contracting+0.15Current ATR < 85% of 10-bar ATR average (volatility decreasing)
        Volume Buildup+0.20Volume increasing 3 consecutive bars AND volume ratio > 1.2 (accumulation)
        OBV Rising+0.10OBV[-1] > OBV[-5] (net buying pressure over 5 bars)
        Near Resistance+0.15Price within 1.5% of a recent swing high (approaching breakout level)
        Neutral RSI+0.05RSI between 43–57 (not extended, room to run)

        Potential Levels

        HIGH = score > 0.60 (strong pre-breakout setup) MEDIUM = score > 0.35 (moderate pre-breakout setup) LOW = score ≤ 0.35 (no compression signal)

        Swing Pivot Detection

        Swing highs and lows are detected using an N-bar pivot rule (N=5): a bar is a swing high if its high is the maximum in the surrounding 5-bar window on both sides. The nearest confirmed swing high/low gives the proximity score.

        Using the Breakout Filter

        Click the 🚀 Breakout button in the scanner header to toggle the breakout filter. When active, only HIGH and MEDIUM potential coins are shown. The "Breakout" column shows the score and icons: 🔥 = BB squeeze active, 📈 = volume buildup detected.

        10. Price Monitor

        A background task checks all OPEN journal entries every 60 seconds and auto-closes them when stop-loss or take-profit levels are reached.

        Hit Detection Logic

        DirectionWIN (TP hit)LOSS (SL hit)
        BULLISH (Long)price ≥ targetprice ≤ stop
        BEARISH (Short)price ≤ targetprice ≥ stop

        Safety Checks

        Before monitoring, the system validates stop placement:

        • BULLISH: stop must be below entry. If not, the entry is skipped with a warning.
        • BEARISH: stop must be above entry. If not, the entry is skipped with a warning.

        PnL Calculation on Close

        pnl_pct = (exit_price − entry) / entry × 100 For BEARISH trades: pnl_pct = −pnl_pct (inverted — profit when price falls) pnl_usdt = position_value × pnl_pct / 100 Capped at: −2 × risk_amount (2R maximum loss)

        When a trade closes, capital is immediately updated, the journal entry is written, and all connected browsers receive a WebSocket push with the new capital state.

        12. Action Plan

        The Action Plan section appears at the top of the Evaluation tab when trades are available. It runs a set of rule-based checks on your metrics and produces prioritized recommendations.

        PriorityConditionAction Suggested
        HIGHWin rate < 45% (≥5 trades)Only take Grade A setups until >55% achieved
        HIGHDrawdown > 8%Cut risk to 0.25% per trade; avoid Grade C
        HIGH4+ consecutive lossesPause 24h; review entry criteria
        HIGHGrade C total PnL < −$50Stop all Grade C trades
        MEDIUMProfit Factor < 1.3Require score ≥ 0.70 and ADX > 25
        MEDIUMGrade A win rate 10%+ higher than Grade BIncrease Grade A allocation
        MEDIUMMore losing months than winningReview MTF alignment discipline
        MEDIUMSharpe < 0.5Improve R:R — target minimum 2.5:1
        Recommendations only appear when there are enough closed trades to be statistically meaningful. With fewer than 5 trades, most rules are suppressed to avoid premature conclusions.

        13. AI Analysis (GPT-4o)

        The AI Analysis section uses OpenAI's GPT-4o model to generate a written assessment of your trading performance. It is optional and requires an OPENAI_API_KEY environment variable.

        What It Analyzes

        • Overall win rate, profit factor, Sharpe ratio
        • Grade-by-grade breakdown (which setups are working)
        • Monthly PnL consistency
        • Probability calibration accuracy
        • Recent 10-trade performance
        • Capital growth and drawdown trajectory

        Caching & Refresh

        The analysis is cached for 6 hours to avoid repeated API calls. It regenerates automatically if the cache is stale, or immediately when you click Generate Analysis. The UI shows the analysis timestamp and a "Refreshing…" state during generation.

        Setup

        Create a .env file in the app directory:

        OPENAI_API_KEY=sk-...
        Without an API key, the AI section shows a "key not configured" message. All other features work without it.
